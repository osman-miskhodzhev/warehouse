# warehouse
Система управления складом на Django 4.2
Данная система предназначена для управления простым складом. На складе могут хранится крупные партии различных 
товаров, которые потом развозятся по розничным магазинам, в по более мелким партиям

Для оприходования поставки в системе необходимо завести приходной ордер.

Для отпуска груза в системе необходимо завести отпускной ордер.

Некоторые corner-кейсы
1. При заполнении приходного ордера может оказаться что фактическое количество товара не совпадает с предполагаемым
Может быть два случая:
Если товара недостаточно, необходимо завести притензию на поставку
Если товара слишком много, то необходимо завести притензию об излишках

2. Кладовщик ошибся при заполнении ордера
Если кладовщик ошибся при заполнении ордера, уже провел его и хочет его редактировать, то он должен оставить заявку менеджеру склада
Заявка - простое электорнное уведомление


В системе должен быть реализован дашборд с:
общей стоймостью всех товаров на складе - сумма закупочных цен всех остатков на складе
количество складских операций за определенный промежуток времени - количество приходов + количество отгрузок
количество приходов и отгрузок по отдельности

## Отчеты
В системе должна быть возможность формировать следующие отчеты:

Ведомость по остаткам
    
    Ведомость по остаткам в системе - документ который содержит информацию о:
    текущих остатках товаров на складе

Отчтет по поставщикам

    Отчет по поставщикам в системе - документ который содержит информацию о всех поставках от конкрентного 
    поставщика в заданный промежуток времени

Отчет о складских операциях

    Отчет о складских операциях в системе - документ содержащий ифнормацию о складских операциях за определенных промежуток времени

Отчет о поставках

    Отчет о поставках в системе - документ содержащий ифнормацию о поставках за определенных промежуток времени

Отчет об отгузках

    Отчет об отгузках в системе - документ содержащий ифнормацию об отгрузках за определенных промежуток времени


В системе должно быть реализовано разграничение доступа к данным по следующим ролям:

    Администратор - полный доступ
    Владелец склада - полный доступ
    Менеджер склада - удаление, редактирование приходных и расходных ордеров
    Кладовщик:

        Добавление и заполнение приходных и расходных ордеров
        Информация на дашборде только об операциях авторизированного кладовщика

## Документы
Приходной ордер в системе - документ предназначеный для хранения информации о:

    дате и времени поставки
    наименовании поставщика
    количестве и виде (видах) товара (товаров)
    общей стоймости поставки

Отпускной ордер в системе - документ предназначеный для хранения информации о:

    дате и времени отпуска
    наименование контрагента
    количестве и виде (видах) товара (товаров)
    общей стоймости отпускаемого груза

Притензия на поставку в системе - документ содержащий информацию о:

    дате и времени поставки
    количестве недостающих единиц
    номер документа по которому сверялось количество товара (траспортная накладная)
    (Данный документ отправляется на почту поставщика)

Притензия об излишках в системе документ содержащий информацию о:

    дате и времени поставки
    количестве лишних единиц
    номер документа по которому сверялось количество товара (траспортная накладная)
    (Данный документ отправляется на почту поставщика)
